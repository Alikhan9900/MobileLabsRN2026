## 🧠 Реалізація навігації

У роботі використано **Expo Router**, де структура папки `app` автоматично визначає маршрути застосунку.

### Основні елементи:

* `_layout.tsx` — кореневий layout
* `(auth)` — група публічних екранів
* `(app)` — група захищених екранів
* `[id].tsx` — динамічний маршрут
* `+not-found.tsx` — екран для неіснуючих сторінок

## 📖 Висновки / Відповіді на контрольні запитання

### 1. Яким чином за допомогою Expo Router реалізується перенаправлення неавторизованого користувача?

Перенаправлення реалізується у файлі `_layout.tsx` захищеної групи маршрутів. Якщо користувач не авторизований, повертається компонент `<Redirect href="/login" />`.

### 2. У чому полягає різниця між використанням компонента `<Link>` та метода `router.push()`?

`<Link>` використовується безпосередньо в JSX для переходу як елемент інтерфейсу, а `router.push()` викликається програмно в коді, наприклад після натискання кнопки або виконання певної логіки.

### 3. Як працюють динамічні маршрути в Expo Router і як отримати передані параметри?

Динамічні маршрути створюються за допомогою файлів у квадратних дужках, наприклад `[id].tsx`. Передані параметри отримуються через хук `useLocalSearchParams()`.

### 4. Чому стан авторизації доцільно зберігати у глобальному контексті, а не в локальному стані компонента?

Глобальний контекст дозволяє використовувати стан авторизації в будь-якому екрані застосунку. Це зручно для перевірки доступу до захищених маршрутів і для централізованого керування входом та виходом користувача.

### 5. Для чого використовуються групи маршрутів `(folderName)` і як вони впливають на URL-адресу?

Групи маршрутів використовуються для логічного розділення екранів, наприклад на публічні й захищені. Назва групи не входить до URL, але допомагає організувати структуру проєкту.
